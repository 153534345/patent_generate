标题title
一种防止NR系统小区节能关闭后出现覆盖盲区的方法及NR系统
摘要abst
本发明提供一种防止NR系统小区节能关闭后出现覆盖盲区的方法及NR系统，所述方法包括：在NG‑RAN NODE CONFIGURATION UPDATE信令中增加NeighbourPowerUp信元；所述NeighbourPowerUp信元用于指示备份小区在主小区节能关闭后通过提升下行发射功率来增加覆盖范围。本发明可以有效解决主小区节能关闭后出现部分覆盖盲区的问题，提高NR系统节能特性的应用场景以及NR系统可靠性。
权利要求书clms
1.一种防止NR系统小区节能关闭后出现覆盖盲区的方法，其特征在于，所述方法包括：在NG-RAN NODE CONFIGURATION UPDATE信令中增加NeighbourPowerUp信元；所述NeighbourPowerUp信元用于指示备份小区在主小区节能关闭后通过提升下行发射功率来增加覆盖范围。2.根据权利要求1所述的防止NR系统小区节能关闭后出现覆盖盲区的方法，其特征在于，所述在NG-RAN NODE CONFIGURATION UPDATE信令中增加NeighbourPowerUp信元的方法为：在NG-RAN NODE CONFIGURATION UPDATE信令中的Served Cells To Modify NR信元里，在Deactivation Indication信元基础上增加一个Neighbour Power Up信元。3.根据权利要求1所述的防止NR系统小区节能关闭后出现覆盖盲区的方法，其特征在于，备份小区需要提升的下行发射功率需要预先计算并配置。4.根据权利要求3所述的防止NR系统小区节能关闭后出现覆盖盲区的方法，其特征在于，计算备份小区需要提升的下行发射功率的方法为：对现网的小区覆盖进行整体规划，计算出备份小区需要提升的下行发射功率，该需要提升的下行发射功率满足完全覆盖主小区节能关闭后导致的覆盖盲区。5.根据权利要求4所述的防止NR系统小区节能关闭后出现覆盖盲区的方法，其特征在于，所述方法包括如下步骤：S1，计算备份小区需要提升的下行发射功率，将计算得到的需要提升的下行发射功率配置给备份小区；S2，在NG-RAN NODE CONFIGURATION UPDATE信令中增加NeighbourPowerUp信元；S3，在节能特性启动时，主小区关闭并向备份小区发送NG-RAN NODE CONFIGURATIONUPDATE信令时，在NG-RAN NODE CONFIGURATION UPDATE信令中需要携带NeighbourPowerUp信元；S4，备份小区收到NG-RAN NODE CONFIGURATION UPDATE信令后，如果在NG-RAN NODECONFIGURATION UPDATE信令中携带了NeighbourPowerUp信元，则根据配置的需要提升的下行发射功率，重新配置本备份小区的下行发射功率。6.根据权利要求5所述的防止NR系统小区节能关闭后出现覆盖盲区的方法，其特征在于，所述方法还包括如下步骤：S5，在节能特性解除时，主小区重新打开并向备份小区发送NG-RAN NODECONFIGURATION UPDATE信令；S6，备份小区收到NG-RAN NODE CONFIGURATION UPDATE信令后，如果发现主小区重新打开，则将本备份小区的下行发射功率恢复到初始值。7.一种NR系统，其特征在于，所述NR系统用于执行权利要求5或6所述防止NR系统小区节能关闭后出现覆盖盲区的方法。
